# ADR 0005: 转换管道与错误处理

## 状态

已采纳

## 背景

需确定批量转换的协调方式和失败处理策略。

## 决策

- **管道**：串行处理。Reader → Document → Writer，一个文件完成后再处理下一个。
- **失败处理**：跳过失败文件，继续处理剩余文件，转换结束后汇总（X 成功 / Y 失败 / Z 跳过）。
- **并行**：不做。PyMuPDF 底层 mupdf 非线程安全，且 PDF 处理为 CPU-bound，Python GIL 限制下多线程无实质加速。

## 后果

- 大文件批量转换时用户等待时间较长，UI 需显示单文件进度 + 总进度
- 转换线程通过 QThread + signal/slot 与 UI 通信，避免主线程阻塞
- 每个文件的转换结果（成功/失败/输出路径）记录在结果列表中
