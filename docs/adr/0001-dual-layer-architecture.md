# ADR 0001: 双层模型架构选型

## 状态

已采纳

## 背景

需要设计一个可扩展的文档转换架构，支持 PDF/Word/PPT/图片之间的相互转换。评估了四种方案：

1. **结构化 AST（Pandoc 路线）**：Haskell ADT 表达文档结构。问题：PDF 没有结构，必须从坐标推断语义，AST 无法承载视觉信息。
2. **纯页面描述模型**：文档 = Page → Element[]，每元素有精确坐标。PDF 转换效果好，但丢失语义（标题层级、阅读顺序），跨域转换（PPT→Word）困难。
3. **双层模型（语义+视觉）**：两种信息分开存储，按目标格式按需取用。适用范围最广，设计复杂。
4. **VLM/LLM**：视觉大模型"看懂"文档再生成输出。不可本地运行，幻觉风险高。

## 决策

采用**方案 3（双层模型）**，但 V1 使用简化版——以视觉层为主，element_type 和 confidence 字段预留语义层扩展点。

## 理由

- 方案 1 对 PDF→Word 场景能力不足，而 PDF→Word 是 V1 核心目标。
- 方案 2 在文档域内转换可工作，但无法扩展到 PPT→Word 等需要语义的场景。
- 方案 4 违反"本地运行"约束。
- 方案 3 在文档域的所有转换对中都有良好表现，是唯一能覆盖全部 V1 目标的方案。
- V1 的简化实现降低初始复杂度，同时架构不锁死，可渐进演进到完整双层模型。

## 后果

- 需要自己设计文档模型数据结构，无可参考的开源实现。
- V1 的简化视觉层与未来完整双层模型之间的迁移路径需要在编码时预留。
- 语义推断（PDF 坐标→段落/标题/表格）是核心技术挑战，质量上限取决于启发式规则的精巧程度。
